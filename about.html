Details and Brainstorming

Fixing what ain't broke

Finding Typos

Not every 404 is actually missing

For us to offer any viable options for mis-spellings/typos, we need to know what resources are available. We need to have a list of available resources to check against, dropping back into directory structure won't work with rewrites; it won't work with variables assigned to files. There needs to be a better option.

Perhaps two:

  1. One, where nothing is done, the operator just lets possible typos slip through
  2. Another where we know most (if not all) of the pages available through a barebones sitemap, created one of two ways:
    1. created by crawling nightly/weekly/monthly,
    2. or created by every page hit.

Creating on page-hit would add overhead to every request, only files that get hit would be matched against (so unpopular files might never get on the list); crawling might be a better alternative.

Then again; seems like making some type or crawler is near annoying; with lots of stuff that can go wrong, and even more things I would have to consider.

Remote Interaction

Giving more than an error

Logging should be remote, asking everyone to create a database, just for tracking errors seems like a lot of overhead. If I offer to log remotely, even with the overhead of parsing another URL and getting a response is far simpler then them having to delegate another database to little ol' me.

Dynamic Results

Based upon logging, and possibly returned responses, the 404.php script needs to be changed a bit:

If they have an account, we can log. Otherwise they're on their own.

If they also track known pages we can offer up some spelling suggestions for mis–spelled requests a lot simpler then dropping back to directory structure for suggestions

They also could setup 410 error codes; or redirection, or a number of other things that we can inform the client about.

Depending upon this, whether they have an account or not, the page should be built based upon data we collect at our server about the request.

This means we simply have to give customers one script, have errors point there, and we can set error codes as we please.
